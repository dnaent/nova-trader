# Backtest Optimization Skill

This skill provides step-by-step instructions for executing, evaluating, and applying optimal parameter changes generated by the weekend GPU backtesting and simulation models.

## Applicability
Activate this skill when:
- The user requests to "run weekend optimizer" or "perform backtesting".
- You need to evaluate Walk-Forward Validation proposals.
- You need to perform Monte Carlo shuffles to stress-test historical trades.
- You want to sync generated parameter diffs back to the main `config.yaml` file.

## Core Workflows

### 1. Running the Backtest Proposer
The entrypoint for generating optimization suggestions is `backtest/proposer.py`.
To execute:
```bash
python3 -m backtest.proposer
```
This script will:
1. Fetch historical trade entries from the SQLite database.
2. Run Walk-Forward Validation on parameter grids (`gate_min`, `exec_threshold`).
3. Run 10,000 Monte Carlo bootstrap shuffles of trade outcomes to compute expected Max Drawdowns.
4. Output a proposals file: `proposed_parameters_<date>.md`.

### 2. Evaluating Proposals
Read the generated markdown proposal file and verify the following safety rules:
- **Drawdown Limit**: Ensure the 99% confidence Worst-Case Drawdown does not exceed the wrapper-level `max_drawdown_pct` (default: 15%).
- **Sharpe Gain**: Verify that the optimized Sharpe ratio target represents a meaningful improvement over the baseline Sharpe.

### 3. Syncing Parameters
If the user approves the optimization proposal:
- Modify `config.yaml` to overwrite `gate_min` and `exec_threshold` with the proposed values.
- Log the update to the system logs.
- Trigger a reboot signal for the active trading engine daemon.
